Gitea Project

Introduction

Le projet Gitea consiste à installer une alternative à GitHub sur un serveur local en utilisant Docker. L'objectif est de connecter les serveurs de développement et de production via un serveur Gitea.

Configuration Docker Compose

Un fichier docker-compose.yaml a été utilisé pour configurer les services nécessaires aux environnements de développement, production, et Gitea.

Configuration de Gitea

Les utilisateurs administrateurs et spécifiques aux environnements de développement et production ont été créés, ainsi que des dépôts initiaux pour gérer les codes sources.

Conclusion

Le projet démontre la puissance de Gitea et Docker pour gérer efficacement des workflows en local, reliant des environnements de développement et de production.
